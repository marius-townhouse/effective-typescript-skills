# Contributing to Effective TypeScript Skills

Thank you for your interest in contributing! This project aims to make TypeScript best practices accessible through AI skills.

## How to Contribute

### Reporting Issues

If you find errors or outdated information:
- Open an issue with a clear description
- Reference the specific skill and book item
- Suggest the correction if possible

### Adding New Skills

1. **Create the skill directory**:
   ```bash
   mkdir skills/your-skill-name
   ```

2. **Follow the skill format** (see existing skills for examples):
   ```markdown
   ---
   name: skill-name
   description: Use when [trigger]. Use when [trigger].
   ---
   
   # Skill Title
   
   ## Overview
   Brief explanation.
   
   ## When to Use This Skill
   - Situation 1
   - Situation 2
   
   ## The Iron Rule
   Core principle.
   
   ## Detection
   How to recognize.
   
   ## Pressure Resistance Protocol
   How to respond to pressure.
   
   ## Red Flags
   Warning signs.
   
   ## Common Rationalizations
   Excuses and rebuttals.
   
   ## Quick Reference
   Summary table.
   
   ## The Bottom Line
   One-sentence summary.
   
   ## Reference
   - Effective TypeScript, 2nd Edition by Dan Vanderkam
   - Item XX: [Full item title]
   ```

3. **Update the README**: Add your skill to the appropriate chapter table

4. **Submit a pull request**:
   - Clear description of what you're adding
   - Reference the book item if applicable
   - Ensure all links work

### Improving Existing Skills

- Fix typos or unclear explanations
- Add better code examples
- Improve the "Pressure Resistance Protocol"
- Clarify the "Iron Rule"

## Style Guidelines

- **Be concise**: Skills should be scannable
- **Use practical examples**: Real-world code beats abstract concepts
- **Be actionable**: Every skill should have clear triggers
- **Cite the book**: Always reference the specific item
- **Use consistent formatting**: Follow existing skill structure

## Code of Conduct

- Be respectful and constructive
- Focus on improving TypeScript practices
- Credit the original book and author
- Help others learn

## Questions?

Open an issue with your question or reach out to the maintainers.

Thank you for contributing!
